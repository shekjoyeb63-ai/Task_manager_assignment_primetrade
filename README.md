# TaskFlow API

A secure, scalable REST API for task management with role-based access control (Employee & Admin), built with Flask, SQLAlchemy, and JWT authentication.

---

## Tech Stack

- **Backend:** Python 3.11+, Flask
- **Auth:** Flask-JWT-Extended, Flask-Bcrypt
- **Database:** SQLite (dev) / PostgreSQL (prod via `DATABASE_URL`)
- **Rate Limiting:** Flask-Limiter
- **CORS:** Flask-CORS
- **Frontend:** Vanilla HTML/CSS/JS â€” _UI designed by AI (Claude by Anthropic); not part of the backend evaluation_

## Setup & Installation

### 1. Clone the repo

```bash
git clone https://github.com/your-username/taskflow-api.git
cd taskflow-api
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

Copy `.env.example` to `.env` and fill in the values:

```bash
cp .env.example .env
```

**.env.example:**

```env
JWT_SECRET_KEY=your-very-secret-key-change-this
DATABASE_URL=sqlite:///Expense.db
```

For PostgreSQL in production:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/taskflow
```

### 5. Run the server

```bash
python Flask_routes.py
```

Server starts at `http://127.0.0.1:5000`

### 6. Run the frontend

Open `index.html` directly in your browser. No build step required.

## API Reference

Base URL: `http://localhost:5000/api/v1`

### Authentication

| Method | Endpoint | Role | Description |
|--------|----------|------|-------------|
| POST | `/register_employee` | Public | Register a new employee |
| POST | `/register_admin` | Public | Register a new admin |
| POST | `/login_employee` | Public | Login as employee, returns JWT |
| POST | `/login_admin` | Public | Login as admin, returns JWT |
| POST | `/logout` | Any (JWT) | Blacklist current token |

All protected routes require the header:
```
Authorization: Bearer <token>
```

### Tasks

| Method | Endpoint | Role | Description |
|--------|----------|------|-------------|
| POST | `/tasks` | Employee | Create a task |
| GET | `/tasks` | Admin | Get all tasks |
| GET | `/my_tasks` | Employee | Get own tasks |
| GET | `/tasks/<id>` | Any (JWT) | Get task by ID |
| PUT | `/tasks/<id>` | Owner or Admin | Update task |
| DELETE | `/tasks/<id>` | Owner or Admin | Delete task |
| GET | `/tasks/filter?status=&priority=` | Any (JWT) | Filter tasks |

#### Task object

```json
{
  "id": 1,
  "title": "Fix login bug",
  "description": "Token expiry not handled",
  "status": "pending",
  "priority": "high",
  "due_date": "2025-05-01",
  "created_at": "2025-04-15",
  "owner_id": 3
}
```

#### Valid values

- `status`: `pending` | `in_progress` | `done`
- `priority`: `low` | `medium` | `high`

## API Testing with Bruno (Recommended)

This project includes a **Bruno collection** for API testing. Bruno is a free, open-source Postman alternative with no account required.

### Install Bruno

Download from [usebruno.com](https://www.usebruno.com/) (available for Windows, macOS, Linux).

### Import the collection

1. Open Bruno
2. Click **Open Collection**
3. Select the `bruno/` folder in this repo

### Set the environment

In Bruno, select the `Local` environment. The base URL is pre-set to `http://localhost:5000/api/v1`.

After logging in, copy the token from the login response and set it in the collection-level **Auth â†’ Bearer Token** field, or use the environment variable `{{token}}`.

### Manual Postman / Insomnia alternative

If you prefer Postman:

1. Create a new collection called **TaskFlow API**
2. Set a collection variable: `base_url = http://localhost:5000/api/v1`
3. Add a collection-level **Bearer Token** auth using variable `{{token}}`
4. Import the requests below or add them manually

#### Quick curl examples

**Register employee:**
```bash
curl -X POST http://localhost:5000/api/v1/register_employee \
  -H "Content-Type: application/json" \
  -d '{"email": "emp@test.com", "password": "pass123"}'
```

**Login (save the token):**
```bash
curl -X POST http://localhost:5000/api/v1/login_employee \
  -H "Content-Type: application/json" \
  -d '{"email": "emp@test.com", "password": "pass123"}'
```

**Create task:**
```bash
curl -X POST http://localhost:5000/api/v1/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"title": "My first task", "priority": "high"}'
```

**Filter tasks:**
```bash
curl "http://localhost:5000/api/v1/tasks/filter?status=pending&priority=high" \
  -H "Authorization: Bearer <token>"
```

```yaml
version: "3.9"
services:
  api:
    build: .
    ports:
      - "5000:5000"
    env_file: .env
    depends_on:
      - db
  db:
    image: postgres:15
    environment:
      POSTGRES_USER: taskflow
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: taskflow
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
```

Set `DATABASE_URL=postgresql://taskflow:secret@db:5432/taskflow` in `.env`.

## Scalability Notes

- **Database:** Swap SQLite for PostgreSQL/MySQL via `DATABASE_URL`. SQLAlchemy abstracts the driver â€” no code changes needed.
- **Token blacklist:** The current `BlacklistedTokens` table works at small scale. For high traffic, replace with **Redis** (`SET jti EX 3600`) to avoid a DB hit on every request.
- **Rate limiting:** Flask-Limiter supports Redis as a backend storage â€” set `RATELIMIT_STORAGE_URL=redis://...` for distributed deployments.
- **Horizontal scaling:** The app is stateless (JWT auth). Run multiple instances behind an **Nginx** or **AWS ALB** load balancer. Shared Redis handles rate limit state.
- **Microservices path:** Auth (register/login/logout) and Task management are already logically separated (`Flask_routes.py` + `Model.py`). They can be split into independent services with an API gateway in front.
- **Caching:** Frequently read endpoints like `GET /tasks` (admin) can be cached with Flask-Caching + Redis, invalidated on any write.
- **Containerization:** Docker-ready. Deploy to AWS ECS, Railway, Render, or any container platform.
